# Data Dictionary - Marketing A/B Testing

## 数据集概述

- **数据来源**: Kaggle - Marketing A/B Testing Dataset
- **数据规模**: 约 588,000 条用户记录
- **时间范围**: 实验期间的用户行为数据
- **实验类型**: 两组对照实验（ad vs psa）

## 字段说明

| 字段名 | 数据类型 | 说明 | 示例值 | 备注 |
|--------|----------|------|--------|------|
| `user id` | Integer | 用户唯一标识符 | 1234567 | 主键，唯一标识每个用户 |
| `test group` | String | 实验分组 | "ad" 或 "psa" | ad=广告组，psa=对照组 |
| `converted` | Boolean | 是否转化 | True / False | True=转化，False=未转化 |
| `total ads` | Integer | 用户看到的广告总数 | 15 | 曝光频次 |
| `most ads day` | String | 看到最多广告的星期几 | "Monday" | 周一到周日 |
| `most ads hour` | Integer | 看到最多广告的小时 | 14 | 0-23 小时制 |

## 字段详细说明

### 1. user id（用户ID）
- **用途**: 唯一标识每个参与实验的用户
- **数据质量要求**: 
  - 不应有缺失值
  - 不应有重复值（每个用户只出现一次）
- **分析用途**: 用于去重、样本量统计

### 2. test group（实验分组）
- **取值**:
  - `ad`: 广告组（实验组），看到商业广告
  - `psa`: 公益组（对照组），看到公益广告
- **分组原则**: 随机分配（需验证）
- **分析用途**: 核心分组变量，用于对比转化率差异

### 3. converted（是否转化）
- **定义**: 用户是否完成目标行为（如点击、购买等）
- **取值**:
  - `True`: 转化成功
  - `False`: 未转化
- **分析用途**: 主要结果指标，计算转化率

### 4. total ads（广告总数）
- **定义**: 用户在实验期间看到的广告总数
- **取值范围**: 正整数
- **分析用途**: 
  - 曝光频次分析
  - 分层变量（低/中/高曝光）
  - 探索曝光频次与转化的关系

### 5. most ads day（最多广告日）
- **定义**: 用户看到最多广告的星期几
- **取值**: Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday
- **分析用途**: 
  - 时段分析
  - 探索周几投放效果更好

### 6. most ads hour（最多广告时）
- **定义**: 用户看到最多广告的小时（24小时制）
- **取值范围**: 0-23
- **分析用途**: 
  - 时段分析
  - 探索哪个时段投放效果更好

## 核心指标定义

### 主指标

#### Conversion Rate（转化率）
- **定义**: 转化用户数 / 总用户数
- **计算公式**: `CR = (converted == True).sum() / total_users`
- **业务意义**: 衡量广告投放效果的核心指标

### 辅助指标

#### Average Total Ads（平均曝光次数）
- **定义**: 用户平均看到的广告数量
- **计算公式**: `mean(total_ads)`
- **业务意义**: 衡量曝光强度

#### Conversion Lift（转化提升）
- **定义**: 实验组相对对照组的转化率提升
- **计算公式**: `(CR_ad - CR_psa) / CR_psa * 100%`
- **业务意义**: 衡量广告投放的增量效果

## 数据质量检查清单

- [ ] 缺失值检查：所有字段是否有缺失
- [ ] 重复值检查：user id 是否唯一
- [ ] 分组均衡性：ad 组和 psa 组样本量是否接近
- [ ] 基础分布一致性：两组的 total_ads、day、hour 分布是否相似
- [ ] 异常值检查：total_ads 是否有异常大的值

## 实验设计假设

### 零假设（H0）
- 广告组（ad）和对照组（psa）的转化率无显著差异
- `CR_ad = CR_psa`

### 备择假设（H1）
- 广告组（ad）的转化率显著高于对照组（psa）
- `CR_ad > CR_psa`（单侧检验）
- 或 `CR_ad ≠ CR_psa`（双侧检验）

## MDE（最小可检测效应）

- **定义**: 业务上认为有意义的最小转化率提升
- **建议设定**: 
  - 保守：≥ 1% 绝对提升
  - 中等：≥ 0.5% 绝对提升
  - 激进：≥ 0.2% 绝对提升
- **用途**: 判断统计显著的结果是否具有业务价值

## 分层分析维度

### 按曝光频次分层
- **低曝光**: total_ads < 25th percentile
- **中曝光**: 25th ≤ total_ads < 75th percentile
- **高曝光**: total_ads ≥ 75th percentile

### 按时段分层
- **工作日 vs 周末**: most_ads_day
- **时段**: most_ads_hour（可分为早/中/晚/夜）

## 注意事项

1. **随机化验证**: 需要验证两组在基础特征上是否均衡
2. **辛普森悖论**: 分层分析时注意整体效应与分层效应的一致性
3. **多重检验**: 进行多个分层分析时需要考虑多重检验校正
4. **样本量**: 确保每个分层有足够的样本量支持统计推断
