# 简历描述写法参考 - Marketing A/B Testing 项目

## 完整版（推荐用于简历）

**投放 A/B 测试与转化率评估（Marketing A/B Testing）**｜个人项目｜2025.03 - 2025.04｜独立完成

- **项目描述**：基于 Kaggle Marketing A/B Testing 数据集（约 58 万用户），评估"广告投放（ad）vs 对照组（psa）"对转化率的影响，输出是否值得投放的统计与业务结论。

- **核心工作**（STAR 内化）：
  - **指标与假设**：定义主指标（conversion rate）与业务阈值（MDE），明确 H0/H1 与检验口径，确保实验结论可解释
  - **数据检查**：对分组样本量、基础分布与异常值进行核查，使用卡方检验验证样本比例失配（SRM），确保实验结果可信
  - **统计推断**：使用两样本比例检验/Bootstrap 估计转化率差异，给出 p-value 与 95% 置信区间，量化效应量（Cohen's h）
  - **Practical Significance**：结合 MDE 判断结果是否具备业务意义，输出投放建议与风险提示，避免"统计显著但效应太小"的误判
  - **分层分析**：按曝光频次与投放时段分层对比效果差异，识别更优人群/时段策略，探索异质性效应

- **技术栈**：Python（pandas, numpy, scipy, statsmodels）, Jupyter Notebook, 统计推断, A/B Testing

- **交付物**：
  - 完整分析 Notebook（代码 + 图表 + 结论）
  - A/B 测试工具函数库（ab_utils.py）
  - 实验结论卡（experiment_summary.md）

### 必须能回答的问题

#### 1. 实验设计类

**Q: 为什么选择这个项目？**
A: 补齐简历中 A/B 测试能力的缺口。这个项目覆盖了实验设计、假设检验、统计推断、业务意义评估等完整流程，非常贴合数据分析岗位对实验分析能力的要求。

**Q: 零假设和备择假设是什么？**
A: 
- H0（零假设）：广告组和对照组的转化率无显著差异，CR_ad = CR_psa
- H1（备择假设）：广告组和对照组的转化率有显著差异，CR_ad ≠ CR_psa（双侧）或 CR_ad > CR_psa（单侧）

**Q: 如何设定 MDE（最小可检测效应）？**
A: MDE 是业务上认为有意义的最小提升。我设定为 [X]%，理由是：
- 参考行业标准（如电商转化率提升 0.5-1% 通常有业务价值）
- 结合成本收益分析（投放成本 vs 转化收益）
- 保守估计，避免过度投放

#### 2. 统计方法类

**Q: 为什么用两样本比例检验？**
A: 因为我们比较的是两个独立样本的转化率（比例），这是典型的两样本比例检验场景。使用 Z 检验基于正态近似，在大样本下（np > 5 且 n(1-p) > 5）是合理的。

**Q: 置信区间如何解释？**
A: 95% 置信区间 [a, b] 表示：如果重复进行相同实验 100 次，约 95 次的置信区间会包含真实的转化率差异。如果区间不包含 0，说明差异显著。

**Q: Bootstrap 的优势是什么？**
A: 
- 不依赖正态分布假设，更稳健
- 适用于小样本或分布未知的情况
- 可以直接从数据中估计抽样分布
- 我用 Bootstrap 作为 Z 检验的补充验证

**Q: 效应量（Cohen's h）是什么？**
A: 效应量衡量差异的实际大小，不受样本量影响。Cohen's h 是专门用于比例差异的效应量：
- |h| < 0.2: 小效应
- 0.2 ≤ |h| < 0.5: 中等效应
- |h| ≥ 0.5: 大效应

#### 3. 业务理解类

**Q: 统计显著 vs 业务意义的区别？**
A: 
- **统计显著**：p-value < 0.05，说明差异不太可能是随机产生的
- **业务意义**：差异是否足够大到值得投入资源去实施
- 大样本下，微小差异也可能统计显著，但业务上可能不值得投放

**Q: 如何判断是否值得投放？**
A: 需要同时满足两个条件：
1. 统计显著（p-value < 0.05）
2. 业务有意义（提升 ≥ MDE）

如果只满足统计显著但效应太小，不建议投放。

**Q: 分层分析发现了什么？**
A: [根据实际分析结果填写，示例]
- 高曝光用户的转化提升更明显（[X]% vs [Y]%）
- 工作日投放效果优于周末（[X]% vs [Y]%）
- 下午时段（12-18点）效果最佳

#### 4. 数据质量类

**Q: 如何检查 SRM（样本比例失配）？**
A: 使用卡方检验验证实际分组比例是否与预期一致：
- 预期比例：50% vs 50%（或其他设定）
- 实际比例：[X]% vs [Y]%
- 卡方统计量：[Z]，p-value：[P]
- 如果 p-value < 0.05，说明存在 SRM，需要检查随机化过程

**Q: 如何验证随机化？**
A: 对比两组在基础特征上的分布：
- 样本量是否接近
- total_ads（曝光频次）分布是否相似
- most_ads_day/hour 分布是否相似
- 如果差异显著，说明随机化可能有问题

**Q: 数据有什么限制？**
A: 
- 数据集来自 Kaggle，可能不是真实业务数据
- 缺少用户画像信息（年龄、性别、地域等），限制了更深入的分层分析
- 缺少成本数据，无法做完整的 ROI 分析
- 时间跨度未知，无法分析时间趋势

## 简历写作注意事项

### ✅ 推荐写法

- 用数据说话：具体的样本量、转化率、p-value、置信区间
- 强调方法论：假设检验、置信区间、效应量、MDE
- 突出业务思维：Practical Significance、分层分析、投放建议
- 展示完整流程：数据检查 → 统计推断 → 业务评估 → 行动建议

### ❌ 避免写法

- 过于简单："做了一个 A/B 测试"（没有细节）
- 过于技术："使用了 scipy.stats.proportions_ztest"（面试官可能不懂）
- 缺少结论："分析了转化率差异"（没有说明结果和建议）
- 夸大其词："显著提升了转化率"（这是分析项目，不是实际业务）
